208

Already Reported

Description

The HTTP 208 Already Reported response status is used in a 207 Multi-Status response to save space and avoid conflicts. If the same resource is requested several times (for example as part of a collection), with different paths, only the first one is reported with 200. Responses for all other bindings will report with this 208 status code, so no conflicts are created and the response stays shorter.
